OCI purpose

  • Multiprocess software architecture
  • DBUS based implementation
  • A central component: the CMAF (Cisco Multi-Application Framework)
  • 3 types of elements: User Interfaces, Services and Data Sources
  • Applications are packaged and declared using a manifest

What is a Package

A file archive used to ship applications into the platform.
It contains:

  • A unique package identifier
    (e.g. "com.cisco.vss.WelcomeMenu")
  • A mandatory manifest
  • Optional icon, libraries, html/css/js files, resources, etc

The OCI SDK

  • A cross-compilation toolchain
  • A customized IDE: Eclipse
  • Supports QtCreator, QtDesigner
  • Some command line tools

Public API overview

  • The User Interfaces which provide the system visual element
  • The Services are background running tasks
  • The Data Sources whose jobs is to provide access the data available on the system
